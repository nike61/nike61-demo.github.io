Стандартные расчеты по времени

Расчет скользящей годовой суммы

Скользящая годовая сумма показывает, сколько бизнес получил за последние 12 месяцев на конец выбранного периода. Она сглаживает сезонность и помогает отслеживать долгосрочный темп продаж без сброса накопления в начале года.

последние 12 месяцев MAT PYMAT рост MAT

Что показывает MAT

Столбцы - продажи по месяцам. Синяя линия - сумма за последние 12 месяцев, зеленая - такое же окно годом ранее. Разница между ними дает рост MAT.

Sales Amount Sales MAT Sales PYMAT

Когда применять

Паттерн полезен, когда нужно видеть устойчивый годовой темп бизнеса в каждой точке времени, а не только сумму текущего месяца или накопление с начала календарного года.

Хороший сценарий

Аддитивные показатели, которые корректно суммируются по датам: выручка, продажи, количество заказов, объем отгрузок. Лучший разрез - день или месяц, где каждая точка показывает итог за последние 12 месяцев.

Лучше выбрать другой расчет

Если бизнесу нужны недели 4-4-5, рабочие дни, нерегулярный финансовый календарь или окно иной длины, таблицу дат и границы окон нужно пересчитать под эту календарную логику.

Идея расчета

Для каждой видимой даты мера строит собственный диапазон дат. Например, MAT для марта включает продажи с апреля прошлого года по март текущего года.

Найти конец периода

Контекст визуализации задает последний видимый день через MAX ( 'Date'[Day Index] ).

Открыть окно MAT

MAT Start Day Index хранит первый день диапазона последних 12 месяцев.

Сдвинуть окно на год

PYMAT Start Day Index и PYMAT End Day Index задают такое же окно годом ранее.

Посчитать рост

Sales MATG возвращает разницу, а Sales MATG % делит ее на Sales PYMAT.

Модель данных

Нужна явная календарная таблица Date с непрерывными датами и связь факта Sales[Order Date] с Date[Date]. Границы скользящих окон лучше материализовать в календаре.

Меры

Все пользовательские меры строятся от Sales Amount. Физическая колонка факта называется Sales[Amount], чтобы имя меры не конфликтовало с именем колонки.

Базовая мера и контроль будущих дат

Sales Amount показывает сумму продаж в текущем фильтре. ShowValueForDates - скрытая техническая мера: она возвращает 1 для периодов, которые начинаются не позже последней даты с продажами, и blank для будущих периодов.

Sales Amount :=
SUM ( Sales[Amount] )

ShowValueForDates :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    1,
    DIVIDE ( 0, 0 )
)
Sales MAT: скользящая годовая сумма

Мера суммирует продажи от начала скользящего годового окна до последней видимой даты. Для марта 2024 это диапазон с 1 апреля 2023 по 31 марта 2024.

Sales MAT :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[MAT Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Sales PYMAT: MAT годом ранее

Sales PYMAT считает скользящую годовую сумму для сопоставимого периода прошлого года. Это база сравнения для абсолютного и процентного роста.

Sales PYMAT :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PYMAT Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PYMAT End Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Sales MATG и Sales MATG %: рост MAT

Sales MATG возвращает разницу между текущим и прошлогодним MAT только тогда, когда обе меры не blank. Sales MATG % показывает эту разницу в процентах от Sales PYMAT.

Sales MATG :=
IF (
    OR (
        ISBLANK ( Sales[Sales MAT] ),
        ISBLANK ( Sales[Sales PYMAT] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales MAT] - Sales[Sales PYMAT]
)

Sales MATG % :=
DIVIDE (
    Sales[Sales MATG],
    Sales[Sales PYMAT]
)
Sales PYMAT (2): альтернативная запись

Смысл альтернативной меры тот же: получить MAT для сопоставимого периода прошлого года. Для стабильной работы она использует те же подготовленные границы окна, что и Sales PYMAT.

Sales PYMAT (2) :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PYMAT Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PYMAT End Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Как не ошибиться в интерпретации

MAT читается как значение на конец выбранного периода. Это не сумма всех MAT-точек внутри года или квартала.

Первые периоды неполные

В первые месяцы после старта данных Sales MAT накапливает доступную историю и может быть меньше полного годового окна. Это ожидаемое поведение для трендового графика.

Рост стабилизируется позже

Sales MATG % может быть очень высоким в начале второго года, потому что прошлогодний MAT еще опирается на короткую историю. Полнее показатель читается после накопления полного предыдущего окна.

Неполный последний год

Если последний год содержит факты только до марта, годовой разрез сравнит доступный текущий результат с полным прошлогодним MAT. Это нужно явно учитывать в подписи или выборе визуализации.

!

На полном закрытом году MAT может совпасть с обычной суммой продаж. Например, окно с 1 января по 31 декабря дает тот же результат, что и Sales Amount за год. На месячном графике смысл другой: каждая точка месяца показывает последние 12 месяцев на конец этого месяца.

Визуальные разрезы

Лучший способ показать паттерн - временной график, где каждая точка имеет понятную конечную дату окна.

Хорошо работает

  • Year Month на оси линейного графика для Sales MAT, Sales PYMAT и Sales MATG %.
  • Date Label для дневного тренда, если данные действительно анализируются по дням.
  • Товар, регион, канал продаж или сегмент клиента как обычные бизнес-фильтры поверх календаря.
  • Годовая таблица для контрольного чтения, если рядом объяснен эффект неполного последнего года.

Может сбивать с толку

  • Разрез по дню недели или признаку рабочего дня, если пользователь ожидает окно только из таких дней. В этом паттерне окно календарное и непрерывное.
  • Суммирование Sales MAT по кварталам или годам как обычной аддитивной меры. MAT уже является агрегатом за окно.
  • Сравнение с будущими месяцами после последней продажи. Такие периоды должны оставаться blank.
  • Использование нестандартного календаря без пересчета колонок MAT Start Day Index, PYMAT Start Day Index и PYMAT End Day Index.

Практические условия реализации

В этой версии паттерна расчетные окна вынесены в таблицу Date. Так меры остаются читаемыми, а поведение дашборда легче проверять.

Окна считаются через индексы. Вместо динамического построения периода внутри меры используются физические колонки календаря. Поэтому точность результата зависит от подготовки Date.

Blank возвращается явно. Для пустого результата используется DIVIDE ( 0, 0 ), чтобы визуализация не получала ноль там, где периода сравнения нет.

Фильтр окна снимает текущие фильтры с календаря. Диапазон строится через FILTER ( ALL ( 'Date' ), ... ). Дополнительные атрибуты календаря нужно применять только если они не меняют смысл окна.

Будущие периоды скрываются отдельно. ShowValueForDates и DateWithSales не дают дашборду продолжать MAT после последней даты с фактами.
