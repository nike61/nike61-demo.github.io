Паттерн показывает, как сравнить текущий год, квартал или месяц с сопоставимым прошлым периодом и не исказить результат, когда последний период еще не завершен.

Честная база сравнения

Если текущий август содержит продажи только до 15 числа, предыдущий месяц, квартал или год тоже обрезается до эквивалентной даты.

Sales Amount Sales PP DateWithSales

Когда применять

Паттерн нужен для отчетов, где пользователь оценивает динамику относительно сопоставимого прошлого периода, а не просто смотрит сумму за выбранные даты.

Хороший сценарий

Аддитивные показатели, которые корректно суммируются по времени: продажи, выручка, количество заказов, объем операций. Модель использует обычный календарь и явную таблицу Date с одной строкой на каждый день.

Лучше выбрать другой подход

Нестандартные недели, календарь 4-4-5, плавающий финансовый календарь, сравнение только по рабочим дням или метрики, которые нельзя сравнивать простой разницей сумм. Для таких правил календарь и формулы нужно проектировать отдельно.

Идея расчета

Текущий период сравнивается с окном той же длины и позиции в прошлом: год назад, квартал назад или месяц назад. Затем считается абсолютный и процентный рост.

Определить текущий контекст

Визуализация задает видимый год, квартал или месяц через атрибуты таблицы Date.

Сдвинуть окно назад

Колонки PY, PQ и PM Day Index хранят границы эквивалентных прошлых периодов.

Обрезать неполный период

DateWithSales не дает сравнивать 15 дней текущего периода с полным предыдущим периодом.

Выбрать нужную меру

Sales POP переключается между YOY, QOQ и MOM по текущей детализации даты.

Модель данных

Для стандартных расчетов нужна непрерывная календарная таблица. Таблица фактов Sales связывается с ней по дате заказа, а расчетные границы прошлых периодов заранее подготовлены в Date.

Колонка Зачем нужна Как формируется
Date[Date] Дневной ключ календаря и сторона связи с Sales[Order Date]. Одна строка на каждый день нужного диапазона без пропусков.
Date Key, Date Label Технический ключ и пользовательская дневная подпись. Date Key хранится как YYYYMMDD, подпись сортируется по ключу.
Year, Year Quarter, Year Month Календарные уровни для матриц, графиков и фильтров. Строятся из даты; текстовые подписи сортируются по техническим ключам.
Year Quarter Key, Year Month Key Ключи сортировки и определения уровня для adaptive POP-мер. Например, 20253 для третьего квартала 2025 года и 202508 для августа 2025.
Day Index Непрерывная числовая шкала дней для фильтра окна. Последовательный номер дня в таблице Date.
DateWithSales Отделяет даты, которые уже могут иметь факты, от будущих дат. 1, если дата не позже последней даты с продажами; иначе 0.
PY Day Index, PQ Day Index, PM Day Index Начальные границы окон предыдущего года, квартала и месяца. Для каждой даты содержат индекс сопоставимой даты в прошлом периоде.

Sales[Order Date], Sales[Order Day Index], Sales[Amount] Дата заказа, ее дневной индекс и сумма продажи. Order Date связан с Date[Date]; Order Day Index повторяет индекс даты заказа.
даты без пропусков связь Sales -> Date григорианские месяцы и кварталы сдвинутые окна в Date

Меры

Физическая колонка факта называется Sales[Amount], а пользовательская мера - Sales Amount. Так имя меры не конфликтует с именем колонки.

Базовая мера и контроль будущих дат

Sales Amount показывает текущую сумму продаж. ShowValueForDates - техническая мера: она возвращает 1, если выбранный период начинается не позже последней даты с продажами, и blank для будущих периодов.

Sales Amount :=
SUM ( Sales[Amount] )

ShowValueForDates :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    1,
    DIVIDE ( 0, 0 )
)
Предыдущие периоды: Sales PY, Sales PQ, Sales PM

Sales PY берет сопоставимый период годом ранее, Sales PQ - в предыдущем квартале, Sales PM - в предыдущем месяце. Все три меры используют заранее подготовленные границы и скрывают будущие даты.

Sales PY :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MIN ( 'Date'[PY Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PY Comparable Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales PQ :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MIN ( 'Date'[PQ Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PQ Comparable Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales PM :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MIN ( 'Date'[PM Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PM Comparable Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Если в модели оставлен контрольный вариант Sales PY (2), он должен совпадать с Sales PY. Для пользовательского отчета обычно достаточно основной меры Sales PY.

Рост к прошлому периоду: YOY, QOQ, MOM

Абсолютный рост считается как текущий период минус предыдущий. Процентный рост делит эту разницу на значение предыдущего периода.

Sales YOY :=
IF (
    OR (
        ISBLANK ( Sales[Sales Amount] ),
        ISBLANK ( Sales[Sales PY] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales Amount] - Sales[Sales PY]
)

Sales YOY % :=
DIVIDE (
    Sales[Sales YOY],
    Sales[Sales PY]
)

Sales QOQ :=
IF (
    OR (
        ISBLANK ( Sales[Sales Amount] ),
        ISBLANK ( Sales[Sales PQ] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales Amount] - Sales[Sales PQ]
)

Sales QOQ % :=
DIVIDE (
    Sales[Sales QOQ],
    Sales[Sales PQ]
)

Sales MOM :=
IF (
    OR (
        ISBLANK ( Sales[Sales Amount] ),
        ISBLANK ( Sales[Sales PM] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales Amount] - Sales[Sales PM]
)

Sales MOM % :=
DIVIDE (
    Sales[Sales MOM],
    Sales[Sales PM]
)
Adaptive POP: Sales PP, Sales POP, Sales POP %

Sales PP возвращает значение предыдущего периода, Sales POP - абсолютный рост, Sales POP % - процентный рост. Выбор годовой, квартальной или месячной логики зависит от того, какой уровень дат остался в текущем контексте.

Sales PP :=
SWITCH (
    TRUE (),
    MIN ( 'Date'[Year Month Key] ) = MAX ( 'Date'[Year Month Key] ), Sales[Sales PM],
    MIN ( 'Date'[Year Quarter Key] ) = MAX ( 'Date'[Year Quarter Key] ), Sales[Sales PQ],
    MIN ( 'Date'[Year] ) = MAX ( 'Date'[Year] ), Sales[Sales PY],
    DIVIDE ( 0, 0 )
)

Sales POP :=
SWITCH (
    TRUE (),
    MIN ( 'Date'[Year Month Key] ) = MAX ( 'Date'[Year Month Key] ), Sales[Sales MOM],
    MIN ( 'Date'[Year Quarter Key] ) = MAX ( 'Date'[Year Quarter Key] ), Sales[Sales QOQ],
    MIN ( 'Date'[Year] ) = MAX ( 'Date'[Year] ), Sales[Sales YOY],
    DIVIDE ( 0, 0 )
)

Sales POP % :=
SWITCH (
    TRUE (),
    MIN ( 'Date'[Year Month Key] ) = MAX ( 'Date'[Year Month Key] ), Sales[Sales MOM %],
    MIN ( 'Date'[Year Quarter Key] ) = MAX ( 'Date'[Year Quarter Key] ), Sales[Sales QOQ %],
    MIN ( 'Date'[Year] ) = MAX ( 'Date'[Year] ), Sales[Sales YOY %],
    DIVIDE ( 0, 0 )
)

Как читать результат

Главный риск в period-over-period анализе - сравнить неполный период с полным. Этот паттерн строит сопоставимое окно, поэтому значение предыдущего периода может быть меньше полного итога месяца, квартала или года.

Первый период остается пустым

У первого года нет предыдущего года, у первого месяца нет предыдущего месяца. В таких строках предыдущий период, рост и процент роста возвращают blank.

Неполный август сравнивается с частью прошлого

Если факты есть до 15 августа, месячное сравнение берет 1-15 июля, квартальное - сопоставимую часть предыдущего квартала, годовое - 1-15 августа прошлого года.

Будущие периоды скрываются

Сентябрь или четвертый квартал не должны показывать хвост предыдущих значений, если фактов там еще нет. Для таких периодов меры возвращают blank.

!

Процент роста читается от предыдущего периода. Sales POP % делит разницу на Sales PP. Если предыдущий период пустой или равен нулю, процент тоже не стоит интерпретировать как обычное число.

Визуальные разрезы

POP-меры хорошо работают, когда визуализация явно показывает календарный уровень: год, квартал или месяц. Дополнительные бизнес-фильтры можно применять поверх календаря.

Хорошо работает

  • Матрица с уровнями Year, Year Quarter, Year Month.
  • Отдельные таблицы или графики для Sales YOY, Sales QOQ и Sales MOM, когда нужен фиксированный тип сравнения.
  • Sales PP, Sales POP и Sales POP % в визуализациях, где выбран ровно один календарный уровень.
  • Фильтры по товару, региону, каналу продаж или клиентскому сегменту, если бизнес ожидает сравнение внутри выбранного сегмента.

Может вводить в заблуждение

  • Итог по нескольким годам без выбранного уровня дат: адаптивные POP-меры возвращают blank, потому что неясно, с каким периодом сравнивать общий итог.
  • Разрез по дню недели, признаку рабочего дня или другой календарной метке, если пользователь ожидает сравнение только внутри этой метки. Окно паттерна остается календарным.
  • Фильтр, который оставляет один месяц внутри годовой строки: адаптивная POP-мера может выбрать месячную логику, потому что видит один Year Month Key.
  • Сравнение нестандартных периодов, которые не совпадают с календарным месяцем, кварталом или годом.

Практические условия реализации

В Visiology-версии паттерна сдвинутые окна материализованы в таблице Date. За счет этого меры остаются читаемыми, а результат стабильно проверяется в дашборде.

Календарь должен быть стандартным. Год начинается 1 января, кварталы - в январе, апреле, июле и октябре, месяц является календарным месяцем.

Фильтр окна снимает текущие фильтры с календаря. Меры строят собственный диапазон через FILTER ( ALL ( 'Date' ), ... ), поэтому календарные атрибуты нужно использовать осознанно.

Blank возвращается явно. Для пустого результата используется DIVIDE ( 0, 0 ). Так визуализация не получает ноль там, где значения быть не должно.

Адаптивный POP не заменяет явный выбор меры. Если отчет всегда сравнивает месяцы, проще и понятнее показать Sales MOM и Sales MOM % напрямую.
