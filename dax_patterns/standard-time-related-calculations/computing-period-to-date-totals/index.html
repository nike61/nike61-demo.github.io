Стандартные расчеты по времени

Computing period-to-date totals

Накопительные итоги показывают, сколько уже набрано с начала года, квартала или месяца. Такой расчет помогает отслеживать прогресс периода, а не только видеть сумму за выбранную дату.

YTD Fiscal YTD QTD MTD будущие периоды скрываются

Как растет итог

Серые столбцы показывают продажи по месяцам. Синяя линия накапливает сумму с начала года, зеленая перезапускается на границе квартала, желтая - на границе месяца.

Sales Amount YTD QTD MTD

Когда применять

Паттерн нужен, когда пользователь сравнивает ход текущего периода с планом, бюджетом или динамикой прошлых периодов и хочет видеть накопленный результат.

Хороший сценарий

Аддитивные показатели, которые корректно суммируются по датам: выручка, продажи, количество заказов, объем операций. Модель использует обычный календарь и явную таблицу Date с одной строкой на каждый день.

Лучше выбрать другой подход

Нестандартные недели, календарь 4-4-5, плавающий финансовый год, расчет только по рабочим дням или периоды, которые не совпадают с календарными месяцами и кварталами. Для таких правил нужно заранее описать собственный календарь.

Идея расчета

Мера заменяет текущий фильтр по датам на диапазон от начала нужного периода до последней даты, видимой в текущем контексте.

Найти конечную дату

Визуальный контекст задает последнюю видимую дату через MAX ( 'Date'[Day Index] ).

Взять начало периода

Начало года, финансового года, квартала или месяца хранится в подготовленных колонках таблицы Date.

Построить окно

FILTER ( ALL ( 'Date' ), ... ) оставляет даты между началом периода и конечной датой.

Скрыть будущие периоды

Если выбранный период начинается после последней даты с продажами, пользовательская мера возвращает blank.

Модель данных

Нужна явная таблица дат с непрерывной дневной шкалой. Таблица фактов Sales связывается с ней по дате заказа.

Колонка Зачем нужна Как формируется
Date Календарная дата и ключ связи с Sales[Order Date]. Одна строка на каждый день нужного диапазона без пропусков.
Date Key, Date Label Техническая сортировка и дневная подпись для визуализаций. Date Key хранится как YYYYMMDD, подпись сортируется по этому ключу.
Year, Year Month, Year Quarter Календарные уровни для таблиц, графиков и фильтров. Строятся из Date; текстовые подписи сортируются техническими ключами.
Fiscal Year Финансовый год для варианта Sales Fiscal YTD. В примере финансовый год начинается 1 июля и заканчивается 30 июня.

Меры

В примерах физическая колонка факта называется Sales[Amount], а пользовательская мера - Sales Amount. Так имя меры не конфликтует с именем колонки.

Базовая мера и контроль будущих дат

Sales Amount показывает сумму продаж. ShowValueForDates - техническая мера: она возвращает 1, если первый день выбранного периода не позже последней даты с продажами, и blank для будущих периодов.

Sales Amount :=
SUM ( Sales[Amount] )

ShowValueForDates :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    1,
    DIVIDE ( 0, 0 )
)
Sales YTD: итог с начала календарного года

Sales YTD (simple) показывает накопление с 1 января до последней видимой даты. Sales YTD добавляет проверку будущих периодов, поэтому сентябрь-декабрь остаются пустыми, если факты закончились в августе.

Sales YTD (simple) :=
CALCULATE (
    SUM ( Sales[Amount] ),
    FILTER (
        ALL ( 'Date' ),
        AND (
            'Date'[Day Index] >= MAX ( 'Date'[Year Start Day Index] ),
            'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
        )
    )
)

Sales YTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Year Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales YTD (2) оставлен как контрольный вариант для календарного YTD. Он должен возвращать те же значения, что и Sales YTD.

Sales YTD (2) :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    TOTALYTD (
        SUM ( Sales[Amount] ),
        'Date'[Date]
    ),
    DIVIDE ( 0, 0 )
)
Sales Fiscal YTD: итог с начала финансового года

Финансовый YTD использует не 1 января, а подготовленную границу Fiscal Year Start Day Index. В примере накопление перезапускается 1 июля.

Sales Fiscal YTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Fiscal Year Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales Fiscal YTD (2) повторяет ту же логику окна как отдельная мера для контрольного сравнения в дашборде.

Sales Fiscal YTD (2) :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Fiscal Year Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Sales QTD: итог с начала квартала

QTD начинает окно с первого дня текущего календарного квартала. На границе 1 апреля, 1 июля и 1 октября накопление начинается заново.

Sales QTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Quarter Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales QTD (2) является отдельной мерой с тем же расчетом. Она нужна, когда в дашборде важно явно сравнить два одинаковых способа расчета.

Sales QTD (2) :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Quarter Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Sales MTD: итог с начала месяца

MTD начинает окно с первого дня текущего месяца. На дневной детализации это обычная накопительная сумма внутри месяца; на уровне месяца это значение на последнюю видимую дату месяца.

Sales MTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Month Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales MTD (2) повторяет расчет как самостоятельная мера для контрольного вывода.

Sales MTD (2) :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Month Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Как читать результат

Period-to-date меры зависят от последней видимой даты. Поэтому один и тот же расчет может выглядеть по-разному на уровне дня, месяца, квартала и года.

Будущие месяцы не продолжают хвост

Sales YTD (simple) может показать итог за год в будущих месяцах. Рабочие меры Sales YTD, Sales QTD и Sales MTD скрывают такие периоды.

День без продаж может иметь накопление

Если день находится до последней даты с фактами, накопительная мера может быть непустой: окно включает предыдущие дни периода.

На родительском уровне важна последняя дата

Годовой или квартальный контекст берет последнюю дату этого периода. Если последний месяц периода находится после фактов, MTD или QTD могут вернуть blank.

!

Проверяйте границы периода. 1 апреля QTD начинается заново, 1 июля в примере начинается новый финансовый год, а 1 число каждого месяца перезапускает MTD. Если календарная таблица подготовлена иначе, смысл меры меняется вместе с ней.

Визуальные разрезы

Лучше всего паттерн читается там, где ось даты сохраняет естественный порядок периода и пользователь понимает, какая дата является конечной для накопления.

Хорошо работает

  • Date Label на дневной оси для MTD и проверки ежедневного накопления.
  • Year Month для YTD и Fiscal YTD, если нужна помесячная картина прогресса года.
  • Year Quarter для QTD и анализа сброса на границах кварталов.
  • Регион, продукт, канал продаж или клиентский сегмент как обычные бизнес-фильтры поверх календаря.

Может исказить чтение

  • Разрез по дню недели или признаку рабочего дня, если ожидается накопление только по выбранным дням. Окно этого паттерна календарное.
  • Итоговая карточка без подписи периода: YTD, QTD и MTD отвечают на разные вопросы и не должны выглядеть как одна и та же сумма.
  • Сравнение будущих периодов с прошедшими. Будущие периоды должны оставаться пустыми, а не повторять последний накопленный итог.
  • Использование финансового YTD без явного Fiscal Year и понятной даты начала финансового года.

Практические условия реализации

В этой версии границы периода материализованы в модели. Такой подход делает формулы короткими, а поведение на дашборде предсказуемым.

Календарь должен быть полным. Если в таблице Date пропущены дни, накопительное окно будет считать не тот диапазон, который ожидает пользователь.

Границы периода задает модель. Year Start Day Index, Quarter Start Day Index и Month Start Day Index должны соответствовать бизнес-календарю.

Blank лучше нуля для будущих периодов. DIVIDE ( 0, 0 ) явно показывает отсутствие значения и не создает ложный ноль.

Дополнительные атрибуты даты требуют проверки. Фильтр окна снимает текущие фильтры с Date через ALL, поэтому разрезы по рабочим дням или дню недели меняют интерпретацию результата.
