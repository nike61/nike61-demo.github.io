Стандартные расчеты по времени

Рост накопленного периода

Паттерн сравнивает накопленную часть текущего года, квартала или месяца с такой же частью прошлого периода: на ту же дату, без сравнения неполного периода с полным.

YOYTD QOQTD MOMTD YTD/QTD/MTD неполные периоды

Сравнение на ту же дату

Если продажи текущего августа есть только до 15 числа, Sales PMTD берет 1-15 июля, а не весь июль.

Sales MTD Sales PMTD DateWithSales

Когда применять

Паттерн нужен, когда бизнесу важен темп движения внутри периода: как год идет относительно прошлого года на ту же дату, как квартал идет относительно предыдущего квартала и как текущий месяц идет относительно прошлого месяца.

Хороший сценарий

Аддитивные показатели, которые корректно суммируются по времени: продажи, выручка, количество заказов, объем операций. Модель использует обычный календарь и явную таблицу Date с одной строкой на каждый день.

Лучше выбрать другой подход

Недельные календари, схема 4-4-5, плавающий финансовый календарь, сравнение только рабочих дней или показатели, где простая сумма с начала периода не является корректной базой для роста.

Идея расчета

Сначала считается текущий накопленный итог: с начала года, квартала или месяца до последней видимой даты. Затем строится такое же окно в периоде сравнения и считается абсолютный и процентный рост.

Найти текущую границу

Визуализация задает последнюю видимую дату через Date: день, месяц, квартал или год.

Собрать to-date окно

Sales YTD, Sales QTD и Sales MTD суммируют продажи от начала периода до текущей даты.

Сдвинуть окно назад

Sales PYTD, Sales PQTD и Sales PMTD берут такую же пройденную часть прошлого периода.

Посчитать рост

YOYTD, QOQTD и MOMTD показывают разницу и процент относительно базы предыдущего накопленного периода.

Модель данных

Для стандартных расчетов нужна непрерывная календарная таблица. Таблица фактов Sales связывается с ней по дате заказа, а границы текущих и прошлых накопленных окон заранее подготовлены в Date.

Колонка Зачем нужна Как формируется
Date[Date] Дневной ключ календаря и сторона связи с Sales[Order Date]. Одна строка на каждый день нужного диапазона без пропусков.
Date Key, Date Label Технический ключ и пользовательская дневная подпись. Date Key хранится как YYYYMMDD, подпись сортируется по ключу.
Year, Year Quarter, Year Month Календарные уровни для матриц, графиков и фильтров. Строятся из даты; текстовые подписи сортируются по техническим ключам.
Year Start Day Index, Quarter Start Day Index, Month Start Day Index Нижние границы текущих окон YTD, QTD и MTD. Для каждой даты содержат индекс первого дня ее года, квартала или месяца.
Day Index Непрерывная числовая шкала дней для фильтра окон. Последовательный номер дня в таблице Date.
DateWithSales Отделяет даты, которые уже могут иметь факты, от будущих дат. 1, если дата не позже последней даты с продажами; иначе 0.
PYTD Comparable Start Day Index, PQTD Comparable Start Day Index, PMTD Comparable Start Day Index Нижние границы окон предыдущего накопленного года, квартала и месяца. Берутся от начала сдвинутого периода, но только для дат, которые можно сравнивать с текущими фактами.
PY Comparable Day Index, PQ Comparable Day Index, PM Comparable Day Index Верхние границы окон предыдущего накопленного периода. Для будущих дат получают техническое низкое значение, чтобы меры вернули пустое значение.
Sales[Order Date], Sales[Order Day Index], Sales[Amount] Дата заказа, ее дневной индекс и сумма продажи. Order Date связан с Date[Date]; Order Day Index повторяет индекс даты заказа.
даты без пропусков связь Sales -> Date григорианские месяцы и кварталы границы to-date в Date

Меры

Физическая колонка факта называется Sales[Amount], а пользовательская мера - Sales Amount. Так имя меры не конфликтует с именем колонки.

Базовая мера и контроль будущих дат

Sales Amount показывает текущую сумму продаж. ShowValueForDates - техническая мера: она возвращает 1, если выбранный период начинается не позже последней даты с продажами, и пустое значение для будущих периодов.

Sales Amount :=
SUM ( Sales[Amount] )

ShowValueForDates :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    1,
    DIVIDE ( 0, 0 )
)
Текущие накопленные итоги: Sales YTD, Sales QTD, Sales MTD

Sales YTD суммирует продажи с начала года, Sales QTD - с начала квартала, Sales MTD - с начала месяца. Все три меры используют последнюю видимую дату текущего контекста и скрывают будущие периоды.

Sales YTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Year Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales QTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Quarter Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales MTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Month Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Previous-to-date база: Sales PYTD, Sales PQTD, Sales PMTD

Sales PYTD берет накопленный итог прошлого года до эквивалентной даты, Sales PQTD - накопленный итог предыдущего квартала, Sales PMTD - накопленный итог предыдущего месяца. Именно эти меры делают сравнение неполного периода корректным.

Sales PYTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PYTD Comparable Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PY Comparable Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales PQTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PQTD Comparable Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PQ Comparable Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales PMTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PMTD Comparable Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PM Comparable Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Контрольная мера Sales PYTD (2), если она оставлена в модели, должна совпадать с Sales PYTD. Для пользовательского отчета обычно достаточно основной меры Sales PYTD.

Рост накопленного периода: YOYTD, QOQTD, MOMTD

Абсолютный рост считается как текущий накопленный итог минус база предыдущего накопленного периода. Процентный рост делит эту разницу на значение этой базы.

Sales YOYTD :=
IF (
    OR (
        ISBLANK ( Sales[Sales YTD] ),
        ISBLANK ( Sales[Sales PYTD] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales YTD] - Sales[Sales PYTD]
)

Sales YOYTD % :=
DIVIDE (
    Sales[Sales YOYTD],
    Sales[Sales PYTD]
)

Sales QOQTD :=
IF (
    OR (
        ISBLANK ( Sales[Sales QTD] ),
        ISBLANK ( Sales[Sales PQTD] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales QTD] - Sales[Sales PQTD]
)

Sales QOQTD % :=
DIVIDE (
    Sales[Sales QOQTD],
    Sales[Sales PQTD]
)

Sales MOMTD :=
IF (
    OR (
        ISBLANK ( Sales[Sales MTD] ),
        ISBLANK ( Sales[Sales PMTD] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales MTD] - Sales[Sales PMTD]
)

Sales MOMTD % :=
DIVIDE (
    Sales[Sales MOMTD],
    Sales[Sales PMTD]
)

Как читать результат

Главная идея в том, что значение предыдущего накопленного периода не обязано совпадать с полным итогом прошлого месяца, квартала или года. Оно совпадает только с уже пройденной частью текущего периода.

Первый период остается пустым

У первого года нет накопленного значения прошлого года, у первого месяца нет накопленного значения прошлого месяца. В таких строках база сравнения, рост и процент роста возвращают пустое значение.

Последний период сравнивается честно

Если факты есть до 15 августа, Sales PYTD берет прошлый год до 15 августа, Sales PQTD - предыдущий квартал до эквивалентного дня, Sales PMTD - 1-15 июля.

День без продаж не ломает накопление

Если внутри текущего периода есть дата без продаж, накопленные меры продолжают показывать накопленный итог за период, пока дата не находится после последней даты фактов.

!

Процент роста считается от базы предыдущего накопленного периода. Sales MOMTD % делит разницу на Sales PMTD, Sales QOQTD % - на Sales PQTD, Sales YOYTD % - на Sales PYTD. Если база пустая или равна нулю, процент не стоит интерпретировать как обычную динамику.

Визуальные разрезы

Меры роста накопленного периода хорошо читаются там, где визуализация явно показывает календарный уровень. Дополнительные бизнес-фильтры можно применять поверх календаря.

Хорошо работает

  • Матрица с уровнями Year, Year Quarter, Year Month и датой.
  • Отдельные таблицы или графики для Sales YOYTD, Sales QOQTD и Sales MOMTD, когда нужен фиксированный тип сравнения.
  • Карточки KPI для текущего года, квартала или месяца, если рядом показана дата, до которой накоплен результат.
  • Фильтры по товару, региону, каналу продаж или клиентскому сегменту, если пользователь ожидает сравнение внутри выбранного сегмента.

Может вводить в заблуждение

  • Общий итог по нескольким годам без понятной последней даты: неясно, какую базу предыдущего накопленного периода должен означать общий результат.
  • Разрез по дню недели, признаку рабочего дня или другой календарной метке, если пользователь ожидает сравнение только внутри этой метки. Окно паттерна остается календарным.
  • Фильтр, который оставляет внутри годовой строки один месяц: пользователь может прочитать годовую строку как YTD, хотя фактически контекст даты уже ограничен месяцем.
  • Сравнение нестандартных периодов, которые не совпадают с календарным месяцем, кварталом или годом.

Практические условия реализации

В Visiology-версии паттерна границы текущих и предыдущих накопленных окон материализованы в таблице Date. За счет этого меры остаются читаемыми и не зависят от неподдерживаемых табличных фильтров на базе функций временной аналитики.

Календарь должен быть стандартным. Год начинается 1 января, кварталы - в январе, апреле, июле и октябре, месяц является календарным месяцем.

Фильтр окна снимает текущие фильтры с календаря. Меры строят собственный диапазон через FILTER ( ALL ( 'Date' ), ... ), поэтому календарные атрибуты нужно использовать осознанно.

Пустое значение возвращается явно. Для пустого результата используется DIVIDE ( 0, 0 ). Так визуализация не получает ноль там, где значения быть не должно.

Поведение на концах месяцев фиксируется в календаре. Сдвиги на месяц и квартал лучше готовить в Date заранее, особенно если в модели есть даты на концах месяцев.
