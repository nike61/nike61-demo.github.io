Стандартные расчеты по времени

Moving averages

Скользящее среднее сглаживает временной ряд и помогает увидеть базовый тренд там, где дневные продажи скачут из-за выходных, промо, крупных разовых заказов или пропусков в фактах.

30 дней 3 календарных месяца 1 год только полные окна

Как читать линию

Серая линия показывает ежедневный факт. Цветные линии показывают среднее за разные скользящие окна: короткое окно реагирует быстрее, длинное сильнее сглаживает шум.

Sales Amount AVG 30D AVG 3M AVG 1Y

Когда применять

Паттерн нужен для отчетов, где важнее направление движения показателя, чем отдельные всплески по дням.

Хороший сценарий

Дневной временной ряд с аддитивной мерой: продажи, выручка, количество заказов, остатки или похожий показатель. Скользящее среднее удобно добавлять на линейный график рядом с базовой мерой.

Лучше выбрать другой расчет

Если окно должно идти по рабочим дням, неделям 4-4-5, пользовательским периодам или нестандартному календарю, модель нужно адаптировать. На уровне месяца результат тоже читается иначе: это значение на последнюю видимую дату периода, а не среднее по месяцам.

Идея расчета

Для каждой видимой даты мера берет окно назад от этой даты, оставляет только даты не позже последней даты с продажами и считает среднее значение базовой меры внутри окна.

Найти текущую дату

Визуальный контекст задает последнюю видимую дату через MAX ( 'Date'[Day Index] ).

Открыть окно

Начало окна хранится в колонках AVG 30D Start Day Index, AVG 3M Start Day Index и AVG 1Y Start Day Index.

Отфильтровать даты

В расчет попадают даты внутри окна и только строки, где DateWithSales = 1.

Показать результат

До накопления полного окна и после последней даты с фактами мера возвращает blank.

Модель данных

Главное условие - явная таблица дат с непрерывным дневным индексом. Она связывается с таблицей фактов Sales по дате заказа.

Колонка Зачем нужна Как формируется
Date Календарная дата и ключ связи с Sales[Order Date]. Одна строка на каждый день без пропусков.
Date Key Технический ключ для сортировки и проверки результата. Целое число в формате YYYYMMDD.
Date Label Подпись даты для таблиц и графиков. Пользовательская строка, отсортированная по Date Key.
Year Month, Year Month Key Месячная подпись и ключ сортировки для агрегированных визуализаций. Месяц даты и числовой ключ месяца.
Day Index Непрерывная шкала дней для фильтра окна. Последовательный номер дня в календаре.
DateWithSales Помогает не показывать значения на будущих датах. 1, если дата не позже последней даты с продажами; иначе 0.
AVG 30D Start Day Index Начало 30-дневного окна. Индекс даты, с которой начинается окно последних 30 дней.
AVG 3M Start Day Index Начало окна последних трех календарных месяцев. Индекс первой даты в трехмесячном окне.
AVG 1Y Start Day Index Начало годового окна. Индекс первой даты в окне последнего года.
все даты без пропусков связь Date -> Sales сортировка подписей через ключи будущие даты скрываются

Меры

Все меры строятся вокруг базовой Sales Amount. В примерах физическая колонка факта называется Sales[Amount], чтобы имя меры не конфликтовало с именем колонки.

Базовая мера и контроль будущих дат

Sales Amount показывает сумму продаж. ShowValueForDates - техническая мера: она возвращает 1 для периодов, которые начинаются не позже последней даты с продажами, и blank для будущих периодов.

Sales Amount :=
SUM ( Sales[Amount] )

ShowValueForDates :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    1,
    DIVIDE ( 0, 0 )
)
Sales AVG 30D: среднее за последние 30 дней

Короткое окно быстрее реагирует на изменение продаж. Мера возвращает blank до первой даты, на которой уже есть полное 30-дневное окно.

Sales AVG 30D :=
IF (
    AND (
        MAX ( 'Date'[DateWithSales] ) = 1,
        CALCULATE (
            MIN ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ) <= MAX ( 'Date'[AVG 30D Start Day Index] )
    ),
    CALCULATE (
        DIVIDE (
            SUM ( Sales[Amount] ),
            DISTINCTCOUNT ( Sales[Order Date] ),
            0
        ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                AND (
                    'Date'[Day Index] >= MAX ( 'Date'[AVG 30D Start Day Index] ),
                    'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
                ),
                'Date'[DateWithSales] = 1
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Sales AVG 3M: среднее за последние 3 месяца

Трехмесячное окно сглаживает недельные и промо-колебания сильнее, чем 30 дней. Это календарные месяцы, а не фиксированные 90 дней.

Sales AVG 3M :=
IF (
    AND (
        MAX ( 'Date'[DateWithSales] ) = 1,
        CALCULATE (
            MIN ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ) <= MAX ( 'Date'[AVG 3M Start Day Index] )
    ),
    CALCULATE (
        DIVIDE (
            SUM ( Sales[Amount] ),
            DISTINCTCOUNT ( Sales[Order Date] ),
            0
        ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                AND (
                    'Date'[Day Index] >= MAX ( 'Date'[AVG 3M Start Day Index] ),
                    'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
                ),
                'Date'[DateWithSales] = 1
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Sales AVG 1Y: среднее за последний год

Годовое окно лучше показывает долгий тренд и сезонность. Оно появляется позже остальных, потому что до первой полной даты годового окна результат скрывается.

Sales AVG 1Y :=
IF (
    AND (
        MAX ( 'Date'[DateWithSales] ) = 1,
        CALCULATE (
            MIN ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ) <= MAX ( 'Date'[AVG 1Y Start Day Index] )
    ),
    CALCULATE (
        DIVIDE (
            SUM ( Sales[Amount] ),
            DISTINCTCOUNT ( Sales[Order Date] ),
            0
        ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                AND (
                    'Date'[Day Index] >= MAX ( 'Date'[AVG 1Y Start Day Index] ),
                    'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
                ),
                'Date'[DateWithSales] = 1
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Как не ошибиться в интерпретации

Скользящее среднее выглядит просто, но смысл результата зависит от окна, календаря и выбранного разреза.

Первые даты скрыты

Если полное окно еще не накопилось, мера возвращает blank. Это лучше, чем показывать среднее по неполному периоду и создавать впечатление резкого старта.

Дни без продаж не всегда ноль

День без факта до последней даты продаж может иметь ненулевое скользящее среднее: окно берет соседние даты с продажами. Это нормально для линии тренда.

После последней продажи blank

Будущие даты не продолжают хвост предыдущих продаж. За это отвечает DateWithSales и техническая логика скрытия будущих периодов.

!

Проверьте знаменатель. В этой реализации сумма в окне делится на DISTINCTCOUNT ( Sales[Order Date] ), то есть на количество дат с продажами в факте. Если бизнесу нужно среднее по всем календарным дням, включая дни с нулем, знаменатель нужно заменить на количество дат календаря внутри окна.

Визуальные разрезы

Лучший визуальный сценарий - линия по датам с несколькими вариантами сглаживания. Остальные разрезы нужно выбирать так, чтобы окно оставалось осмысленным.

Хорошо работает

  • Date Label на дневной оси линейного графика.
  • Year Month для обзорного тренда, если читатель понимает, что точка месяца - значение на последнюю видимую дату.
  • Товар, регион, канал продаж или клиентский сегмент как обычные бизнес-фильтры поверх даты.
  • Сравнение Sales Amount, Sales AVG 30D, Sales AVG 3M и Sales AVG 1Y на одном графике.

Может вводить в заблуждение

  • Разрез по дню недели или признаку рабочего дня, если пользователь ожидает среднее только по таким дням. Окно в этом паттерне календарное и непрерывное.
  • Столбчатые итоги по году или кварталу без пояснения: мера не суммируется за период, а показывает среднее на выбранной конечной дате.
  • Сравнение с неполными будущими периодами. Такие даты должны оставаться blank.
  • Использование паттерна для нестандартного календаря без пересчета стартовых индексов окон.

Практические условия реализации

В Visiology-версии паттерна скользящие окна заранее материализованы в таблице Date. Это делает меру прозрачной для пользователя и стабильной для дашборда.

Окна считаются через индексы. Вместо динамического построения набора дат внутри меры используются колонки стартового индекса. Поэтому качество расчета зависит от того, как подготовлена таблица дат.

Blank возвращается явно. Для пустого результата используется DIVIDE ( 0, 0 ). Так визуализация не получает фиктивный ноль там, где значения быть не должно.

Фильтр окна снимает текущие фильтры с календаря. Мера строит собственный диапазон дат через FILTER ( ALL ( 'Date' ), ... ). Поэтому дополнительные атрибуты календаря нужно применять осознанно.

Имена мер отделены от колонок. Физическая колонка называется Sales[Amount], а пользовательская мера - Sales Amount. Это избавляет модель от конфликта имен.
