Стандартные расчеты по времени

Накопленный период против полного предыдущего периода

Паттерн показывает, когда текущий результат с начала года, квартала или месяца догоняет полный результат прошлого периода. Это удобно, когда прошлый период нужен как фиксированный ориентир.

YTDOPY QTDOPQ MTDOPM полный предыдущий период контроль будущих дат

Фиксированный ориентир

Текущий MTD растет день за днем и сравнивается со всем прошлым месяцем. Процент становится положительным только после пересечения ориентира.

текущий to-date полный предыдущий период момент пересечения

Когда применять

Используйте этот паттерн, если бизнесу важно не просто сравнить два одинаковых отрезка времени, а понять прогресс к полному результату прошлого периода.

Хороший сценарий

Аддитивные показатели, которые корректно суммируются по времени: продажи, выручка, количество заказов, объем операций. Модель использует обычный календарь, а таблица Date содержит одну строку на каждый день.

Лучше выбрать другой подход

Недельные календари, схема 4-4-5, плавающие финансовые периоды, сравнение только рабочих дней и показатели, где накопительная сумма не отражает бизнес-смысл: например средняя цена, маржинальность или конверсия без отдельной логики числителя и знаменателя.

Идея расчета

Текущий период берется с начала года, квартала или месяца до последней видимой даты. Предыдущий период берется целиком: весь прошлый год, весь прошлый квартал или весь прошлый месяц.

Найти текущий итог

Sales YTD, Sales QTD и Sales MTD считают продажи с начала текущего периода.

Взять полный benchmark

Sales PYC, Sales PQC и Sales PMC возвращают весь предыдущий год, квартал или месяц.

Посчитать разницу

YTDOPY, QTDOPQ и MTDOPM показывают, сколько осталось до ориентира или насколько он уже превышен.

Показать процент

Процентные меры делят разницу на полный предыдущий период. Ноль означает, что текущий итог ровно достиг прошлый полный результат.

Модель данных

Расчет опирается на физические границы периодов в таблице Date. Это делает формулы явными и не зависит от функций, которые автоматически сдвигают календарные периоды.

Колонка Зачем нужна Как формируется
Date[Date] Дневной ключ календаря и сторона связи с Sales[Order Date]. Одна строка на каждый день диапазона, без пропусков и дублей.
Day Index Непрерывная числовая шкала дней для фильтрации окон. Последовательный номер дня в таблице Date.
Year, Year Quarter, Year Month Календарные уровни для строк таблиц, осей графиков и фильтров. Строятся из даты; текстовые подписи сортируются по техническим ключам.
Fiscal Year, Fiscal Year Quarter Финансовые уровни, если бизнес сравнивает fiscal year-to-date. В примере финансовый год заканчивается 30 июня; для другой границы нужны свои ключи и подписи.
Year Start Day Index, Quarter Start Day Index, Month Start Day Index Начало текущих окон YTD, QTD и MTD. Для каждой даты хранят индекс первого дня ее года, квартала или месяца.
Fiscal Year Start Day Index Начало текущего финансового года для Sales Fiscal YTD. Рассчитывается по принятой границе финансового года.
PYC Start/End Day Index Границы полного предыдущего календарного года. Для любой даты 2025 года указывают полный 2024 год.
Fiscal PYC Start/End Day Index Границы полного предыдущего финансового года. Для финансового года, который закончится 30 июня 2025 года, база - 1 июля 2023 года - 30 июня 2024 года.
PQC Start/End Day Index, PMC Start/End Day Index Границы полного предыдущего квартала и месяца. Для даты в Q2 база PQC - весь Q1; для даты в апреле база PMC - весь март.
DateWithSales Отделяет даты, которые уже относятся к истории фактов, от будущих дат. 1, если дата не позже последней даты с продажами; иначе 0.
Sales[Order Date], Sales[Order Day Index], Sales[Amount] Дата заказа, ее дневной индекс и сумма продажи. Order Date связан с Date[Date]; Order Day Index повторяет индекс даты заказа.
даты без пропусков связь Sales -> Date стандартные месяцы и кварталы границы предыдущих полных периодов

Меры

Физическая колонка факта называется Sales[Amount], а пользовательская мера - Sales Amount. В формулах ниже пустое значение возвращается через DIVIDE ( 0, 0 ).

Базовая мера и контроль будущих дат

ShowValueForDates - техническая мера: она разрешает показывать результат, если первый день выбранного периода не позже последней даты с продажами. Для будущих периодов она возвращает пустое значение.

Sales Amount :=
SUM ( Sales[Amount] )

ShowValueForDates :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    1,
    DIVIDE ( 0, 0 )
)
Текущие накопленные итоги: Sales YTD, Sales QTD, Sales MTD

Эти меры суммируют продажи от начала текущего года, квартала или месяца до последней видимой даты. В финансовом варианте начало года берется из отдельной колонки.

Sales YTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Year Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales Fiscal YTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Fiscal Year Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales QTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Quarter Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales MTD :=
IF (
    MIN ( 'Date'[Day Index] )
        <= CALCULATE (
            MAX ( Sales[Order Day Index] ),
            REMOVEFILTERS ( 'Date' ),
            REMOVEFILTERS ( Sales )
        ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Month Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)
Year-to-date over full previous year

Sales PYC возвращает продажи за полный предыдущий календарный год. Sales YTDOPY показывает абсолютное отклонение текущего YTD от этой базы, а Sales YTDOPY % - процент относительно базы.

Sales PYC :=
IF (
    AND (
        MIN ( 'Date'[Day Index] )
            <= CALCULATE (
                MAX ( Sales[Order Day Index] ),
                REMOVEFILTERS ( 'Date' ),
                REMOVEFILTERS ( Sales )
            ),
        MIN ( 'Date'[Year] ) = MAX ( 'Date'[Year] )
    ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PYC Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PYC End Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales YTDOPY :=
IF (
    OR (
        ISBLANK ( Sales[Sales YTD] ),
        ISBLANK ( Sales[Sales PYC] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales YTD] - Sales[Sales PYC]
)

Sales YTDOPY % :=
DIVIDE (
    Sales[Sales YTDOPY],
    Sales[Sales PYC]
)
Fiscal year-to-date over full previous fiscal year

Финансовый вариант использует те же принципы, но база берется из полного предыдущего финансового года. Границы финансового года должны быть заранее подготовлены в Date.

Sales Fiscal PYC :=
IF (
    AND (
        MIN ( 'Date'[Day Index] )
            <= CALCULATE (
                MAX ( Sales[Order Day Index] ),
                REMOVEFILTERS ( 'Date' ),
                REMOVEFILTERS ( Sales )
            ),
        MIN ( 'Date'[Fiscal Year Sort] ) = MAX ( 'Date'[Fiscal Year Sort] )
    ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[Fiscal PYC Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[Fiscal PYC End Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales Fiscal YTDOPY :=
IF (
    OR (
        ISBLANK ( Sales[Sales Fiscal YTD] ),
        ISBLANK ( Sales[Sales Fiscal PYC] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales Fiscal YTD] - Sales[Sales Fiscal PYC]
)

Sales Fiscal YTDOPY % :=
DIVIDE (
    Sales[Sales Fiscal YTDOPY],
    Sales[Sales Fiscal PYC]
)
Quarter-to-date over full previous quarter

Sales PQC возвращает весь предыдущий квартал. Поэтому Sales QTDOPQ % становится положительной только после того, как текущий QTD превысит полный квартальный ориентир.

Sales PQC :=
IF (
    AND (
        MIN ( 'Date'[Day Index] )
            <= CALCULATE (
                MAX ( Sales[Order Day Index] ),
                REMOVEFILTERS ( 'Date' ),
                REMOVEFILTERS ( Sales )
            ),
        MIN ( 'Date'[Year Quarter Key] ) = MAX ( 'Date'[Year Quarter Key] )
    ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PQC Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PQC End Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales QTDOPQ :=
IF (
    OR (
        ISBLANK ( Sales[Sales QTD] ),
        ISBLANK ( Sales[Sales PQC] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales QTD] - Sales[Sales PQC]
)

Sales QTDOPQ % :=
DIVIDE (
    Sales[Sales QTDOPQ],
    Sales[Sales PQC]
)
Month-to-date over full previous month

Sales PMC возвращает весь предыдущий месяц. В дневной детализации это помогает увидеть дату, когда текущий MTD догнал или превысил прошлый месяц.

Sales PMC :=
IF (
    AND (
        MIN ( 'Date'[Day Index] )
            <= CALCULATE (
                MAX ( Sales[Order Day Index] ),
                REMOVEFILTERS ( 'Date' ),
                REMOVEFILTERS ( Sales )
            ),
        MIN ( 'Date'[Year Month Key] ) = MAX ( 'Date'[Year Month Key] )
    ),
    CALCULATE (
        SUM ( Sales[Amount] ),
        FILTER (
            ALL ( 'Date' ),
            AND (
                'Date'[Day Index] >= MAX ( 'Date'[PMC Start Day Index] ),
                'Date'[Day Index] <= MAX ( 'Date'[PMC End Day Index] )
            )
        )
    ),
    DIVIDE ( 0, 0 )
)

Sales MTDOPM :=
IF (
    OR (
        ISBLANK ( Sales[Sales MTD] ),
        ISBLANK ( Sales[Sales PMC] )
    ),
    DIVIDE ( 0, 0 ),
    Sales[Sales MTD] - Sales[Sales PMC]
)

Sales MTDOPM % :=
DIVIDE (
    Sales[Sales MTDOPM],
    Sales[Sales PMC]
)

Как читать результат

Отрицательное значение означает, что текущий накопленный итог еще ниже полного прошлого периода. Положительное значение означает, что прошлый полный результат уже повторен или превышен.

Год

Sales YTDOPY % около нуля показывает момент, когда текущий год с начала января достиг продаж за весь прошлый год. До этого процент отрицательный, после пересечения - положительный.

Квартал

Sales QTDOPQ % сравнивает текущий квартал с полным предыдущим кварталом. На уровне года эта база не имеет одного очевидного квартала, поэтому мера должна быть пустой.

Месяц

Sales MTDOPM % хорошо работает на месяце и на днях внутри месяца. Дневная детализация показывает, в какой день текущий месяц догнал полный предыдущий месяц.

Визуальные разрезы

У предыдущего полного периода должен быть однозначный смысл. Поэтому уровень детализации выбирают под конкретную меру, а не смешивают несколько календарных уровней в одном сравнении.

Разрез Когда уместен Что проверить
Year Для Sales YTDOPY и Sales YTDOPY %. В контексте должен быть один календарный год; первый год данных обычно не имеет предыдущего полного года.
Fiscal Year Для Sales Fiscal YTDOPY, если бизнес работает по финансовому году. Граница финансового года должна совпадать с принятой в компании логикой.
Year Quarter Для Sales QTDOPQ и процента к полному предыдущему кварталу. На годовом итоге квартальная база неоднозначна; показывайте кварталы отдельными строками или точками.
Year Month, Date Label Для Sales MTDOPM и поиска даты, когда MTD пересек полный прошлый месяц. Дневная детализация до последней даты фактов может показывать накопленный итог даже в день без продаж.
Товар, канал, регион, клиент Как обычные бизнес-фильтры поверх временного расчета. Фильтр должен одинаково применяться к текущему и предыдущему периоду, иначе сравнение меняет смысл.
День недели, рабочий день, произвольный диапазон дат Только если это осознанно заложено в модель и пересчитаны границы сравнения. Стандартная формула сравнивает с полным предыдущим периодом; дополнительные календарные фильтры могут сделать результат трудным для интерпретации.

Важные edge cases

Эти ситуации стоит проверить до публикации дашборда, потому что внешне они похожи на ошибки, но часто являются ожидаемым поведением паттерна.

Первый период без базы

Первый год, первый квартал или первый месяц в модели не имеет предыдущего полного периода. Меры базы и роста должны возвращать пустое значение.

Неполный текущий период

Неполный 2025 год сравнивается со всем 2024 годом. В начале периода результат обычно сильно отрицательный: это не падение к той же дате, а прогресс к полной прошлой базе.

Будущие даты

Периоды, которые начинаются после последней даты с фактами, скрываются. При этом день без продаж до последней даты фактов может показывать накопленный итог.

Нулевая база

Если полный предыдущий период равен нулю или пустой, процентная мера должна оставаться пустой. Абсолютная разница без базы тоже теряет практический смысл.

!

Главная проверка смысла: пользователь должен понимать, что YTDOPY, QTDOPQ и MTDOPM отвечают на вопрос "догнали ли мы весь прошлый период?", а не на вопрос "как мы идем относительно той же даты прошлого периода?". Для второго вопроса нужен другой паттерн.
