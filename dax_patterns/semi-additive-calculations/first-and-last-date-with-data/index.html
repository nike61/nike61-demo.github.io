Полуаддитивные расчеты

First and last date with data

Паттерн показывает значение показателя на первой или последней дате выбранного периода, для которой в данных есть хотя бы один снимок состояния. Для всего периода выбирается единая дата, поэтому итоговый результат не является суммой последних дат отдельных клиентов.

Главное правило

В Q4 2020 последняя дата с данными - 18 ноября. На этой общей дате есть строка только у Maurizio Macagno, поэтому Luis Bonifaz не получает значение от 15 ноября.

есть строка факта дата расчета

Какую задачу решает

Баланс счета, складской запас, лимит или численность сотрудников описывают состояние на дату. Их можно складывать по клиентам, счетам или подразделениям, но нельзя складывать по всем датам месяца или квартала.

Баланс на доступной дате

Если на календарной границе периода нет снимка, мера берет первую или последнюю дату внутри периода, на которой в факте есть данные.

Одна дата для периода

Внутри строки отчета выбирается одна общая дата. После этого значения на этой дате складываются по клиентам или другим измерениям.

Понятный итог

Итог квартала или года заново ищет свою дату с данными. Он не суммирует месяцы и не складывает последние даты каждого клиента.

Когда применять

Этот вариант подходит для разреженных снимков состояния, когда бизнес хочет видеть последнюю опубликованную дату периода, но дата должна быть общей для всей строки отчета.

Хороший сценарий

Факт хранит балансы только на даты загрузки или закрытия, а отчет за месяц, квартал или год должен показывать состояние на ближайшей дате с данными внутри выбранного периода.

Нужен другой вариант

Если каждый клиент, счет или склад должен иметь собственную последнюю дату, используйте расчет по сущности. Если нужно переносить последнее известное значение вперед, это тоже отдельное бизнес-правило.

Идея расчета

Мера сначала находит дату с данными в текущем контексте календаря, затем применяет ее как фильтр к таблице Date и суммирует строки факта только на этой дате.

Получить период

Строка отчета задает набор дат: месяц, квартал, год, произвольный диапазон или общий итог.

Найти дату с данными

Balance Dates оставляет только даты, где есть хотя бы одна строка баланса в этом периоде.

Оставить одну дату

LastDateWithData берет максимальный Date Key, а FirstDateWithData берет минимальный.

Сложить баланс

На выбранной общей дате суммируются строки Balances[Amount] в текущем разрезе клиента или другого измерения.

Модель данных

Для паттерна нужна непрерывная таблица дат, факт со снимками состояния и техническая таблица дат, где действительно есть данные. Эта таблица фиксирует правило выбора общей даты.

Таблица Назначение Ключевые поля
Date Непрерывный календарь, который задает месяцы, кварталы, годы и общий отчетный диапазон. Date, Date Key, подписи периода и sort-ключи для корректной сортировки.
Customer Справочник клиентов или владельцев счетов. По этому измерению баланс можно складывать, если строки относятся к одной дате. Customer ID, Customer.
Balances Факт со снимками баланса на конкретную дату для конкретного клиента. Date, Date Key, Customer ID, Amount.
Balance Dates Техническая таблица с одной строкой на дату, где есть хотя бы один баланс. Она помогает выбирать дату независимо от клиента. Date, Date Key.
Balances[Date] -> Date[Date] Balances[Customer ID] -> Customer[Customer ID] Balance Dates[Date] -> Date[Date] Balance Dates не связана с Customer

Меры

Обе меры используют один прием: берут минимальный или максимальный Date Key из Balance Dates, затем через FILTER ( ALL ( 'Date' ), ... ) оставляют в расчете только найденную дату.

Balance LastDateWithData

Показывает сумму балансов на последней дате с данными в текущем контексте Date. Дата не зависит от клиента, а финальная сумма сохраняет клиентский фильтр.

Balance LastDateWithData :=
CALCULATE (
    SUM ( Balances[Amount] ),
    FILTER (
        ALL ( 'Date' ),
        'Date'[Date Key] = MAX ( 'Balance Dates'[Date Key] )
    )
)
Balance FirstDateWithData

Показывает сумму балансов на первой дате с данными в текущем контексте. Логика та же, но выбирается минимальный Date Key из Balance Dates.

Balance FirstDateWithData :=
CALCULATE (
    SUM ( Balances[Amount] ),
    FILTER (
        ALL ( 'Date' ),
        'Date'[Date Key] = MIN ( 'Balance Dates'[Date Key] )
    )
)
!

Важно: Balance Dates должна содержать ровно те даты, которые бизнес считает датами с данными. Если строка факта без утвержденного статуса или с нулевым балансом не должна участвовать в поиске даты, это правило лучше учесть при подготовке таблицы.

Как читать результат

Пустое значение означает, что на выбранной общей дате у этой строки отчета нет баланса. Это не ноль и не ошибка суммирования.

Контекст Что выбирает мера Balance FirstDateWithData Balance LastDateWithData
CY 2020, все клиенты Первая дата с данными - 2020-01-31, последняя - 2020-11-18. 4,657.00 1,850.00
CY 2020, Katie Jordan Дата все равно общая для года: 2020-01-31 и 2020-11-18. На 18 ноября у Katie Jordan строки нет. 1,687.00 blank
Q4 2020, Luis Bonifaz Первая дата с данными квартала - 2020-10-31, последняя - 2020-11-18. Личная дата Luis 2020-11-15 не используется. 1,960.00 blank
Q4 2020, Maurizio Macagno На первой и последней датах квартала у клиента есть строки факта. 2,000.00 1,850.00
December 2020, все клиенты В декабре нет ни одной даты с данными, поэтому обе меры возвращают blank. blank blank

Разрезы в дашборде

Паттерн лучше всего читается, когда пользователь видит период и понимает, что все клиенты в одной строке отчета проверяются на одной и той же дате.

Хорошо работают

  • Месяц, квартал и год из таблицы Date с правильной сортировкой.
  • Клиент, счет, склад или подразделение, если бизнес согласен читать их на общей дате периода.
  • Таблица, где рядом показаны Balance FirstDateWithData и Balance LastDateWithData: так видно, как меняется выбранная дата.

Требуют пояснения

  • Сравнение клиентов с разной регулярностью загрузки: blank показывает отсутствие строки на общей дате, а не нулевой баланс.
  • Общий итог страницы: он выбирает дату с данными во всем доступном контексте, а не суммирует видимые строки.
  • Разрезы по измерениям, которых нет в Balance Dates: дата будет общей для этих разрезов, пока helper-таблица не подготовлена на нужной гранулярности.

Важные edge cases

Перед публикацией отчета стоит согласовать именно смысл даты с данными. У похожих требований могут быть разные корректные формулы.

Не персональная дата

Для CY 2020 последняя дата с данными - 2020-11-18. Luis Bonifaz имеет баланс 1,813.00 на 2020-11-15, но годовая мера Balance LastDateWithData для него остается blank.

Период без снимков

Если в выбранном месяце или квартале нет ни одной строки в Balance Dates, первая и последняя даты с данными не находятся, а результат остается пустым.

Правило "есть данные"

Если дата должна считаться доступной только после закрытия периода, проверки качества или утверждения статуса, это нужно отразить в составе Balance Dates.

!

Практическое правило: используйте этот паттерн, когда вопрос звучит как "какой баланс был на общей первой или последней дате периода, где вообще есть данные". Если вопрос звучит как "какой последний баланс каждого клиента", нужен расчет по клиенту.
